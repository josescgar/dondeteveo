# Design System

## UX Direction

- Mobile-first
- Clear, fresh, amateur-sport tone
- Prioritize comprehension over decoration
- Avoid generic dark/purple default aesthetics

## Content Tone

- Friendly, plain, and direct
- Reduce race-day confusion
- Write for non-technical spectators as well as runners

## Core UI Priorities

- Make race search obvious
- Make race-local timing impossible to misunderstand
- Keep share pages readable on small screens
- Keep route and timing information visually grouped

## Visual Tokens To Define In Implementation

- Typography scale
- Color roles
- Spacing scale
- Radius scale
- Shadow scale
- Motion rules

## Component Priorities

- Search input
- Filters
- Buttons
- Form fields
- Cards/lists for races
- Share summary panel
- Map side panel or bottom sheet

## Accessibility Baseline

- Semantic HTML first
- Visible focus states
- Clear labels
- Keyboard-friendly controls
- Good contrast
- Do not rely on color alone for meaning

## Time UX Rule

- Show only race local time
- Always label timezone clearly near predicted times
- Never show ambiguous bare times
